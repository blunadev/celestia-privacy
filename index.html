Data we collect Authentication information: a session token issued by our auth server after Discord OAuth login. Discord user ID (received during OAuth and stored on the server for role verification). How we use data To verify that a user belongs to the required Discord server and role. To allow access to the extension’s features. Data storage The extension stores a session token locally in the browser. The auth server stores session data (including refresh tokens) to allow automatic re‑authentication. Data sharing We do not sell or transfer user data to third parties. Data is only used for authentication and access control. Retention Session tokens expire automatically (within 1 day) and are refreshed when valid.
